Travelly – Overview

  • Type: UX/UI + Front-End Development Project
  • Role: UX Designer, UI Designer, Front-End Developer
  • Tools: Figma, HTML, SCSS, JavaScript, Bootstrap
  • Timeline: 3 weeks
  • Deliverables: High-fidelity prototype, responsive website
Travelly is a responsive travel booking platform designed to make trip planning faster and less overwhelming. Most existing travel sites are crowded and confusing, often forcing users to jump between multiple platforms just to plan one trip. I set out to create a cleaner, more focused experience that allows users to book flights, hotels, and rental cars in one place. I completed this project over several weeks as part of my web design coursework, handling both UX design and front-end development.

Purposes and Goals

The main goal of Travelly was to simplify the booking process and reduce unnecessary distractions. I wanted to give users a single, clear entry point to start their trip planning and make the entire process feel seamless across devices. By focusing on visual hierarchy and mobile-first design, I aimed to remove friction from an experience that is often stressful.

Design Decisions

I began with a competitive analysis of major travel platforms, which revealed cluttered layouts and weak mobile experiences. To address this, I created wireframes that emphasized clarity and clean grouping of content, then built high-fidelity mockups in Figma.

I used a calm, travel-inspired color palette with sky blues and soft neutrals, paired with a modern sans-serif typeface for readability. The layout uses Bootstrap’s grid system, card-based results, and a sticky navigation bar to support a smooth three-step booking flow.

Development and Technology

The site was built using HTML, SCSS, Bootstrap, and JavaScript. SCSS variables and mixins kept the design consistent and flexible, while Bootstrap ensured responsiveness across breakpoints. I also incorporated accessibility best practices, using semantic HTML, ARIA labels, and proper focus states to support keyboard navigation.

Challenges and Solutions

One of the biggest challenges was maintaining a clean interface without sacrificing functionality. By applying progressive disclosure and carefully prioritizing key actions, I created a layout that guided users rather than overwhelming them. Ensuring mobile responsiveness required testing across multiple devices, but the grid system and SCSS variables made adjustments more manageable.

Results

The final product is a clean, fully responsive website that simplifies travel planning. Early feedback highlighted how easy it was to navigate and complete bookings compared to other platforms. This project strengthened my ability to balance clarity, functionality, and accessibility in a real-world scenario.
